# AdTruck — Отчёт: рынок, смета и план MVP до прода

_Реклама на бортах фур (TIR) + live-карта Европы. Рынок — Польша. Статус: интерактивный POC — **https://ad-truck.vercel.app**. Дата: июль 2026._

---

## 1. Резюме
- **Готового «коробочного» софта**, который можно лицензировать и запустить как свою фирму по рекламе на фурах, **на рынке нет.** Есть операторы-сети (продают кампании, не лицензию) и отдельные «кубики» (GPS, измерение, карты), которые подписываешь и встраиваешь.
- Значит наш путь — **строить своё, подключая готовые кубики** — правильный. Это свободная ниша для регионального игрока в PL.
- **Бюджет для семейной фирмы реалистичен:** при работе «ты + AI» — порядка **$1.5–5k разово + $60–250/мес**, а не «миллионы».
- **Главные риски** — не код, а: реальный GPS-трекинг (кооперация перевозчиков), **RODO/GDPR (нужен юрист)**, платежи/бухгалтерия, двусторонний cold-start.

## 2. Рынок: конкуренты и что можно переиспользовать

### 2.1. Аналоги/операторы (у них покупают кампании, лицензию — нельзя)
| Платформа | Что / носитель | Регион | Модель | Для нас |
|---|---|---|---|---|
| **Adverttu → Drovo** | Маркетплейс бренды↔владельцы авто/фургонов/**фур**; приложение, атрибуция, ретаргет | UK/EU | Оплата за кампанию | **Ближайший аналог модели** — эталон фич |
| **Movia Media** | Мобильные билборды **на бортах фур**, GPS, дашборд | US/CA | Оплата за кампанию | Ближайший по носителю |
| **Wrapify / Carvertise** | Обёртка личных авто (rideshare), трекинг | US | Кампании; водителю $100–450/мес | Экономика/метрики |
| **AdQuick** | Маркетплейс OOH + мобильные билборды: GPS-verify, proof-of-play фото, CPM/ROAS | US | Медиабаинг | **Референс дашборда** |
| **Firefly** | Цифровые экраны на такси/rideshare | US | Кампании | Пример LED-ветки на будущее |

### 2.2. «Кубики» — их можно подписать и встроить
- **Реальный GPS:** приложение водителя (₽0, сделаем сами) · **Traccar** (open-source, ~$6–20/мес) · Geotab (~$30–40/авто/мес + железо) · Samsara ($27–33/авто/мес, контракт 36 мес).
- **Измерение показов:** **StreetMetrics** (движущаяся наружка, US/EU, SaaS, цена по запросу) — «сертифицированные» цифры вместо нашей OTS-оценки. На старте оставляем свою OTS (отраслевая норма).
- **Карты/трафик:** MapLibre + OpenFreeMap (уже, бесплатно) + **TomTom/HERE Traffic** (free tier) — для «heatmap по трафику».
- **Программатик-DOOH** (только при цифровых экранах): Vistar, Hivestack (есть white-label), Broadsign.

**Вывод:** GPS = приложение водителя/Traccar; карты/трафик = MapLibre + TomTom; измерение = наша OTS → StreetMetrics позже. Дорогой enterprise-стек на старте не нужен.

## 3. Смета до прода (семейная фирма, «строим с AI»)
| Сценарий | Кэш разово | В месяц | Срок |
|---|---|---|---|
| **Ты + AI (максимум DIY)** | **$1.5–5k** | $60–250 | 2–4 мес part-time |
| + точечный фриланс (платежи, RODO, бэкенд, submission) | +$3–10k | — | 2–3 мес |
| Небольшая студия под ключ (ставки EE/PL $25–50/ч) | $30–70k | — | 3–6 мес |

_Западное агентство: $120–300k+. «Миллионы» — это мульти-рыночные платформы, не наш случай._

**Стек:** **Expo (React Native)** → iOS + Android + web из одного кода (переносим готовую логику: симулятор, оценщик, карту, дизайн).
**Инфра:** managed (рекомендую) — Supabase/Neon + Vercel + Expo EAS ≈ **$25–60/мес**; AWS (как вариант) — Fargate/Lightsail + RDS + S3 + CloudFront + Cognito ≈ **$60–200/мес**.
**Ежемес. подписки:** Apple $99/год, Google $25 разово, домен ~$15/год, карты/трафик $0–50, GPS $0–20, email/SMS $5–30, платежи — комиссия ~1.5–2.9%.

## 4. Подводные камни (честно)
1. **Реальный GPS** — переход от симулятора: приложение водителя (батарея, согласие, фоновая геолокация) или телематика. Сложность **операционная** — кооперация перевозчиков.
2. **RODO/GDPR** — трекинг ТС/людей, данные водителей, ретаргет → политика приватности, согласия, DPA. Техчасть — на нас, **юрист — обязателен** в EU.
3. **App Store/Play review** — фоновая геолокация → строгая модерация (особенно Apple). Проходимо при чётком consent.
4. **«Реальные» показы** — наша OTS — оценка (норма отрасли); «сертифицированные» цифры → платный StreetMetrics.
5. **Cold-start** — нужны реальные фуры до того, как рекламодатели увидят ценность. **Бизнес-риск.**
6. **Масштаб live-карты** — реальную телеметрию тянет бэкенд-ingest + вебсокеты + кластеризация (не клиентский симулятор).
7. **Платежи/VAT (PL)** — Stripe/PayU + faktury → бухгалтер.
8. **Физическая операционка** — печать/оклейка/монтаж, окна доступности фур.

**Хард-блокеров «не сможем закодить» нет.** Весь софт делаем сами; внешние нужны: **юрист (RODO), бухгалтер (VAT), аккаунты сторов, опц. security-ревью и платные данные (трафик/измерение).**

## 5. План: вертикальный срез прод-версии (MVP)
Цель — превратить POC в **настоящий продукт** на минимальном, но полном «вертикальном срезе».

**Стек:** Expo (iOS/Android/web) + **Supabase** (auth + БД Postgres + storage + RLS) + переиспользование текущего TS-кода (домен/симулятор/оценщик/UI).

**В срез входит:**
1. **Auth + роли** (рекламодатель / перевозчик / админ), приглашения, сброс.
2. **Multi-tenant** — изоляция данных по клиенту (RLS-политики Supabase).
3. **Перенос UI** POC в Expo (экраны уже спроектированы).
4. **Заказ кампании по стране / по конкретной фуре** + калькулятор (расширяем оценщик).
5. **Приложение водителя** — реальный GPS вместо симулятора (первый honest-трекинг).
6. **Мини-админка** — клиенты/юзеры/кампании/фуры, модерация.

**Этапы:**
1. Каркас Expo + общий пакет домена (переиспользуем логику) + Supabase-клиент.
2. Auth + роли + схема БД + RLS (multi-tenant).
3. Перенос ключевых экранов (лендинг/визард/карта/кампании).
4. Заказ по стране/фуре + калькулятор на серверных данных.
5. Приложение водителя (геолокация) → живая карта на реальных данных.
6. Мини-админка + деплой (EAS + web) + RODO-минимум.

**Что нужно от клиента:** аккаунт Supabase (или AWS), Apple/Google dev-аккаунты, юрист для RODO, решение по онлайн-платежам.

## 6. Рекомендация / следующий шаг
Начинаем **Тир 1 (ты + AI)** с вертикального среза на **Expo + Supabase**, переиспользуя готовый код. Первый шаг — выбрать бэкенд (Supabase рекомендую) и завести аккаунт, дальше я разворачиваю каркас, auth и multi-tenant.
