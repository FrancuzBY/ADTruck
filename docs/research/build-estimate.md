# AdTruck до прода: объём, бюджет, инфраструктура, подводные камни

_Оценка для небольшой семейной фирмы (рынок PL). Сценарий: строим **вместе со мной (AI)**, не нанимаем агентство за «мировые» деньги. Цены — ориентиры (USD, ≈EUR/PLN), не оффер. Июль 2026._

## Допущения
- **Стек:** один кодовый база — **Expo (React Native)** → iOS + Android + **web** из одного проекта (переносим уже написанную логику: симулятор, оценщик, карта, дизайн). Это самый дешёвый способ закрыть «мобилка + веб».
- **Модель труда:** основную разработку делаешь ты + я (Claude). Значит основной «расход» — твоё время, а **кэш-расходы** — это аккаунты, инфра, подписки и точечный фриланс на дыры.
- **Инфра:** ты просил AWS — дам вариант на AWS, но для семейной фирмы **дешевле и быстрее** управляемый стек (Supabase/Neon + Vercel + Expo EAS). Оба варианта ниже.

## Объём работ до прода
1. **Мультиарендность (multi-tenant):** клиенты (рекламодатели), перевозчики, роли, изоляция данных.
2. **Аутентификация:** email/пароль + OAuth, сброс, приглашения, роли (admin/клиент/перевозчик).
3. **Админ-панель:** управление клиентами, юзерами, фурами, кампаниями, креативами; модерация/одобрения; выставление счетов.
4. **Новый функционал (из твоего списка):**
   - заказ показа **по стране** / **по конкретной фуре**;
   - workflow **изготовления и установки** рекламы с учётом **когда фура свободна** (доступность/расписание);
   - **калькулятор стоимости** (у нас уже есть оценщик — расширить);
   - оплата/счета, отчёты, уведомления.
5. **Реальный трекинг:** приложение водителя (Expo) шлёт GPS, **или** интеграция телематики (Traccar) — вместо симулятора.
6. **Бэкенд/БД/API**, файловое хранилище (креативы/фото), пуш-уведомления.
7. **Прод-обвязка:** домен, SSL, аккаунты App Store/Play, аналитика, мониторинг, бэкапы, приватность (RODO).

## Бюджет — три сценария

| Сценарий | Кэш (единоразово) | Ежемесячно | Срок |
|---|---|---|---|
| **1. Ты + я (AI), максимум DIY** | **$1.5k–5k** (аккаунты, домен, юрист/RODO, немного дизайна) | **$60–250** | 2–4 мес part-time |
| **2. DIY + точечный фриланс** (платежи, усиление бэкенда, submission в сторы, RODO) | **+$3k–10k** | как выше | 2–3 мес |

_Для сравнения: западное агентство за такой объём — $120k–300k+. «Миллионы» — это уже мульти-рыночные платформы, не наш случай._

## Ежемесячная инфраструктура/подписки (мелкий масштаб)
- Apple Developer **$99/год**, Google Play **$25 единоразово**, домен **~$15/год**.
- **Хостинг:**
  - Managed (рекомендую): Supabase/Neon (БД+auth+storage) + Vercel (веб) + Expo EAS (сборки) ≈ **$25–60/мес**.
  - AWS (как просил): Lightsail/ECS-Fargate + RDS + S3 + CloudFront + Cognito ≈ **$60–200/мес** на старте (гибче, но дороже/сложнее в поддержке для семейной фирмы).
- Карты/трафик: MapLibre+OpenFreeMap **бесплатно**; TomTom/HERE Traffic — free tier → **$0–50**.
- GPS: приложение водителя **$0**, или Traccar VPS **~$10–20**.
- Email/SMS (SES/Twilio) **$5–30**; пуши (Expo) **$0**.
- Платежи: Stripe/PayU — без абонплаты, комиссия ~1.5–2.9% с транзакции.

## Подводные камни (честно, что рискованно)
1. **Реальный GPS — главный переход от POC.** Симулятор → реальные координаты: приложение водителя (батарея, согласие, стабильность фоновой геолокации) или телематика (железо, установка, кооперация перевозчиков). Технически решаемо; сложность — **операционная** (уговорить перевозчиков ставить/делиться).
2. **RODO/GDPR (EU) — серьёзно.** Трекинг ТС/людей, данные водителей, ретаргет «оффлайн→онлайн» → нужны политика приватности, согласия, DPA. Мы напишем техчасть, но **нужен юрист** (это gating-пункт в PL/EU).
3. **App Store/Play review.** Фоновая геолокация в приложении водителя → строгая модерация (особенно Apple): нужен чёткий обоснованный consent-флоу. Проходимо, но это известный барьер.
4. **«Реальные» показы vs OTS.** Наши цифры — отраслевая эвристика (это норма для оценок). «Сертифицированные» цифры для клиентов → платный StreetMetrics (US/EU, enterprise). На старте оставляем OTS + heatmap по трафику (TomTom).
5. **Двусторонний cold-start.** Нужны реальные фуры (перевозчики) раньше, чем рекламодатели увидят ценность. **Бизнес-риск, не технический.**
6. **Масштаб live-карты.** Клиентский симулятор не потянет реальный стрим телеметрии — в проде нужен бэкенд-ingest + вебсокеты + кластеризация маркеров. Инженерно решаемо.
7. **Платежи/бухгалтерия PL (VAT, faktury).** Настройка Stripe/PayU + учёт — нужен бухгалтер.
8. **Физическая операционка.** Печать/оклейка/монтаж плёнок, окна доступности фур — это ops-процессы; софт их поддержит, но данные о доступности должны реально приходить от перевозчиков.

## Что мы (ты + AI) точно сделаем сами vs где нужен человек
- **Сделаем сами:** все приложения (iOS/Android/web на Expo), админ-панель, auth/роли, multi-tenant, API/БД, live-карта, калькулятор, workflow заказа/изготовления/установки, heatmap/ETA/отчёты, интеграции карт/трафика/GPS (Traccar/приложение водителя), Stripe-интеграция.
- **Нужен внешний человек/сервис:**
  - **Юрист** — RODO/условия/согласия (обязательно в EU);
  - **Бухгалтер** — VAT/faktury;
  - **Аккаунты и ревью** Apple/Play (сделаем вместе, но аккаунты/оплата на них);
  - **Опц. security-ревью** перед приёмом платежей/PII;
  - **Опц. платные данные** — StreetMetrics/HERE, если нужны «сертифицированные» цифры;
  - **Операционка** — договориться с перевозчиками, печать/монтаж плёнок.
- **Хард-блокеров «не сможем закодить» — нет.** Все риски — операционные/юридические/данные, а не «технически невозможно».

## Рекомендация (по этапам)
1. **MVP-прод (Тир 1):** Expo (iOS/Android/web) на текущем функционале + auth + multi-tenant + админка + заказ по стране/фуре + калькулятор + приложение водителя для реального GPS. Managed-инфра. RODO-минимум с юристом. → 2–4 мес, кэш ~$2–5k, ~$60–150/мес.
2. **Прод v2:** платежи, workflow изготовления/установки с доступностью фур, отчёты/атрибуция, TomTom-трафик, мониторинг/бэкапы.
3. **Позже:** StreetMetrics (сертиф. показы), программатик-DOOH при цифровых бортах, масштабирование live-карты.

_Точные цифры уточним под твои вводные (сколько фур на старте, нужны ли онлайн-платежи сразу, есть ли у перевозчиков телематика, регион запуска)._
